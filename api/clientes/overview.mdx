## Overview

The Clientes (Customers) API allows you to manage customer records in the Softwart system. Customers are the core entity that connects to appointments (Citas) and sales (Ventas).

## Entity Schema

### Cliente Model

| Field | Type | Required | Unique | Description |
|-------|------|----------|--------|-------------|
| `id_cliente` | number | Auto | Yes | Primary key, auto-generated |
| `tipoDocumento` | string | Yes | No | Document type (e.g., DNI, RUC, Pasaporte) |
| `documento` | string | Yes | Yes | Customer's document number |
| `nombre` | string | Yes | No | Customer's full name |
| `correo` | string | Yes | Yes | Customer's email address |
| `telefono` | string | No | No | Customer's phone number |
| `estado` | boolean | Yes | No | Active/inactive status (default: true) |

### Relationships

- **One-to-Many with Citas**: A customer can have multiple appointments
- **One-to-Many with Ventas**: A customer can have multiple sales

<Note>
Deleting a customer is only allowed if there are no associated Citas or Ventas. The API will return a 409 Conflict error if dependencies exist.
</Note>

## Authentication

<Warning>
All Cliente endpoints require authentication and role-based authorization.
</Warning>

**Required Roles**: `Admin` or `Empleado`

**Headers**:
```json
{
  "Authorization": "Bearer YOUR_JWT_TOKEN"
}
```

## Endpoints

### List All Customers

```http
GET /api/clientes
```

Retrieves all customers in the system.

**Response** (200 OK):
```json
{
  "success": true,
  "data": [
    {
      "id_cliente": 1,
      "tipoDocumento": "DNI",
      "documento": "12345678",
      "nombre": "Juan Pérez",
      "correo": "juan.perez@example.com",
      "telefono": "987654321",
      "estado": true
    }
  ]
}
```

### Get Customer by ID

```http
GET /api/clientes/:id
```

Retrieves a specific customer by their ID.

**Parameters**:
- `id` (path parameter): Customer ID

**Response** (200 OK):
```json
{
  "success": true,
  "data": {
    "id_cliente": 1,
    "tipoDocumento": "DNI",
    "documento": "12345678",
    "nombre": "Juan Pérez",
    "correo": "juan.perez@example.com",
    "telefono": "987654321",
    "estado": true
  }
}
```

**Error Response** (404 Not Found):
```json
{
  "success": false,
  "message": "Cliente no encontrado"
}
```

### Create Customer

```http
POST /api/clientes
```

Creates a new customer record.

**Request Body**:
```json
{
  "tipoDocumento": "DNI",
  "documento": "12345678",
  "nombre": "Juan Pérez",
  "correo": "juan.perez@example.com",
  "telefono": "987654321",
  "estado": true
}
```

**Required Fields**: `tipoDocumento`, `documento`, `nombre`, `correo`

**Response** (201 Created):
```json
{
  "success": true,
  "message": "Cliente creado exitosamente",
  "data": {
    "id_cliente": 1,
    "tipoDocumento": "DNI",
    "documento": "12345678",
    "nombre": "Juan Pérez",
    "correo": "juan.perez@example.com",
    "telefono": "987654321",
    "estado": true
  }
}
```

**Error Response** (400 Bad Request):
```json
{
  "success": false,
  "message": "Campos requeridos: tipoDocumento, documento, nombre, correo"
}
```

### Update Customer

```http
PUT /api/clientes/:id
```

Updates an existing customer. Only provided fields will be updated.

**Parameters**:
- `id` (path parameter): Customer ID

**Request Body** (all fields optional):
```json
{
  "tipoDocumento": "RUC",
  "documento": "20123456789",
  "nombre": "Juan Pérez Empresa SAC",
  "correo": "empresa@example.com",
  "telefono": "987654321"
}
```

**Response** (200 OK):
```json
{
  "success": true,
  "message": "Cliente actualizado",
  "data": {
    "id_cliente": 1,
    "tipoDocumento": "RUC",
    "documento": "20123456789",
    "nombre": "Juan Pérez Empresa SAC",
    "correo": "empresa@example.com",
    "telefono": "987654321",
    "estado": true
  }
}
```

### Delete Customer

```http
DELETE /api/clientes/:id
```

Deletes a customer. Only allowed if there are no associated appointments or sales.

**Parameters**:
- `id` (path parameter): Customer ID

**Response** (200 OK):
```json
{
  "success": true,
  "message": "Cliente eliminado correctamente"
}
```

**Error Response** (409 Conflict):
```json
{
  "success": false,
  "message": "No se puede eliminar: existen Cita asociados (5)"
}
```

### Toggle Customer Status

```http
PATCH /api/clientes/:id/estado
```

Toggles the customer's active/inactive status.

**Parameters**:
- `id` (path parameter): Customer ID

**Response** (200 OK):
```json
{
  "success": true,
  "message": "Cliente activado",
  "data": {
    "estado": true
  }
}
```

## Usage Examples

### Creating a Customer with cURL

```bash
curl -X POST https://api.softwart.com/api/clientes \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "tipoDocumento": "DNI",
    "documento": "12345678",
    "nombre": "María García",
    "correo": "maria.garcia@example.com",
    "telefono": "987654321"
  }'
```

### Fetching Customer with JavaScript

```javascript
const response = await fetch('https://api.softwart.com/api/clientes/1', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const result = await response.json();
if (result.success) {
  console.log('Customer:', result.data);
}
```

### Updating Customer Status

```javascript
const response = await fetch('https://api.softwart.com/api/clientes/1/estado', {
  method: 'PATCH',
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const result = await response.json();
console.log(result.message); // "Cliente activado" or "Cliente inactivado"
```

## Error Responses

All endpoints may return the following error responses:

| Status Code | Description |
|-------------|-------------|
| 400 | Bad Request - Missing required fields |
| 401 | Unauthorized - Missing or invalid token |
| 403 | Forbidden - Insufficient permissions |
| 404 | Not Found - Customer doesn't exist |
| 409 | Conflict - Cannot delete due to dependencies |
| 500 | Internal Server Error |

**Generic Error Format**:
```json
{
  "success": false,
  "message": "Error description",
  "error": {} // Additional error details (dev mode only)
}
```
