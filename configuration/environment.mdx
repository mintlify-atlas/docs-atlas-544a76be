## Overview

The Softwart Backend uses environment variables for configuration management. Variables are loaded using the `dotenv` package and should be defined in a `.env` file at the project root.

## Loading Environment Variables

Environment variables are loaded at application startup:

```typescript
import dotenv from "dotenv";
dotenv.config();
```

Reference: `src/app.ts:1-2`, `src/data-source.ts:8-9`

## Required Variables

### Server Configuration

| Variable | Default | Description | Example |
|----------|---------|-------------|----------|
| `PORT` | `3000` | Server port number | `3000` |
| `NODE_ENV` | - | Environment mode | `development`, `production` |

```typescript
const PORT = Number(process.env.PORT ?? 3000);
const isProd = process.env.NODE_ENV === "production";
```

Reference: `src/app.ts:14`, `src/data-source.ts:29`

### Database Configuration

| Variable | Default | Description | Example |
|----------|---------|-------------|----------|
| `DB_HOST` | `localhost` | PostgreSQL host | `localhost`, `db.example.com` |
| `DB_PORT` | `5432` | PostgreSQL port | `5432` |
| `DB_USER` | `root` | Database username | `postgres` |
| `DB_PASSWORD` | `""` | Database password | `your_secure_password` |
| `DB_NAME` | `mi_base_de_datos` | Database name | `softwart_db` |

```typescript
export const AppDataSource = new DataSource({
  type:        "postgres",
  host:        process.env.DB_HOST     ?? "localhost",
  port:        Number(process.env.DB_PORT ?? 5432),
  username:    process.env.DB_USER     ?? "root",
  password:    process.env.DB_PASSWORD ?? "",
  database:    process.env.DB_NAME     ?? "mi_base_de_datos",
});
```

Reference: `src/data-source.ts:38-42`

### Security Configuration

| Variable | Default | Description | Example |
|----------|---------|-------------|----------|
| `JWT_SECRET` | `dev_secret_cambiame_en_prod` | JWT signing secret | `your_random_secret_key_here` |
| `CORS_ORIGIN` | `*` | Allowed CORS origins | `https://yourdomain.com` |

```typescript
const JWT_SECRET = process.env.JWT_SECRET ?? "dev_secret_cambiame_en_prod";
```

Reference: `src/middlewares/auth.middleware.ts:4`, `src/controllers/AuthController.ts:10`

<Warning>
  Never use the default JWT secret in production. Generate a strong random secret key.
</Warning>

## Example .env File

### Development

```bash
# Server
PORT=3000
NODE_ENV=development

# Database
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your_password
DB_NAME=softwart_dev

# Security
JWT_SECRET=dev_secret_cambiame_en_prod
CORS_ORIGIN=http://localhost:5173
```

### Production

```bash
# Server
PORT=3000
NODE_ENV=production

# Database (Supabase/Render)
DB_HOST=db.your-project.supabase.co
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your_secure_production_password
DB_NAME=postgres

# Security
JWT_SECRET=generate_a_strong_random_secret_here
CORS_ORIGIN=https://yourdomain.com
```

## Environment-Specific Behavior

The application adjusts its behavior based on `NODE_ENV`:

### Development Mode
- Database synchronization enabled
- SQL query logging enabled
- Relaxed CORS policy
- Default fallback values for missing variables

### Production Mode
- Database synchronization disabled (use migrations)
- SQL query logging disabled
- Strict CORS policy
- SSL required for database connections
- No default values (all variables required)

## CORS Configuration

CORS origin is configured via the `CORS_ORIGIN` environment variable:

```typescript
export const corsMiddleware: RequestHandler = (req, res, next) => {
  res.setHeader("Access-Control-Allow-Origin", process.env.CORS_ORIGIN ?? "*");
  res.setHeader("Access-Control-Allow-Methods", "GET,POST,PUT,PATCH,DELETE,OPTIONS");
  res.setHeader("Access-Control-Allow-Headers", "Content-Type,Authorization");
  if (req.method === "OPTIONS") {
    res.sendStatus(204);
    return;
  }
  next();
};
```

Reference: `src/middlewares/cors.middleware.ts:3-12`

## Generating JWT Secret

Generate a secure JWT secret using Node.js:

```bash
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

Or using OpenSSL:

```bash
openssl rand -hex 64
```

## Best Practices

<Tip>
  - Never commit `.env` files to version control
  - Add `.env` to your `.gitignore` file
  - Use different secrets for development and production
  - Provide a `.env.example` file with dummy values
  - Validate required environment variables at startup
  - Use strong random secrets for JWT_SECRET
  - Restrict CORS_ORIGIN to your frontend domain in production
</Tip>
